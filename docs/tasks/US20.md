**Тест-кейсы:**
## Acceptance Criteria 1: В интерфейсе склада отображаются остатки ингредиентов с пороговыми значениями. При достижении порога строка выделяется цветом.

### Test case 40. Отображение остатков и пороговых значений
**Тип:** Базовый  
**Предусловие:** Работник авторизован и открыл интерфейс "Склад".  

**Шаги:**  
1. Просмотреть таблицу остатков.  

**OP:**  
1. В таблице есть колонки: "Ингредиент", "Остаток", "Порог", "Статус".  
2. Для ингредиента "Котлета" отображается остаток 50 шт, порог 20 шт.  
3. Для ингредиента "Булочка" отображается остаток 15 шт, порог 20 шт. Строка с "Булочкой" выделена красным цветом, так как остаток ниже порога.  

### Test case 41. Отсутствие порогового значения (некорректные данные)
**Тип:** Негативный  
**Предусловие:** В системе для ингредиента "Сыр" не задано пороговое значение (или оно равно 0).  

**Шаги:**  
1. Открыть интерфейс склада.  
2. Найти в таблице ингредиент "Сыр".  

**OP:**  
1. Ингредиент "Сыр" отображается в списке.  
2. Ингредиент не подсвечиваться как критичный, т. к. порог не задан

## Acceptance Criteria 2: Можно сформировать заявку на пополнение (интеграция с ERP или ручной заказ).

### Test case 42. Создание заявки на пополнение
**Тип:** Базовый  
**Предусловие:** Ингредиент "Булочка" ниже порога.  

**Шаги:**  
1. Открыть интерфейс склада.  
2. Найти в таблице ингредиент "Булочка".  
3. Нажать кнопку "Создать заявку" в его строке.  
4. Задать количество  
5. Подтвердить заявку  

**OP:**  
1. Сообщение "Заявка № 123 отправлена".
