**Тест-кейсы:**
## Acceptance Criteria 1: В интерфейсе менеджера доступен список активных заказов с возможностью поиска по номеру или отображения заказов, ожидающих выдачи.

### Test case 48. Просмотр списка активных заказов и поиск по номеру
**Тип:** Базовый  
**Предусловие:** Менеджер авторизован в системе и имеет доступ к интерфейсу управления заказами. В системе есть несколько заказов в статусе "Ожидает выдачи" (например, №101, №102, №105) и один заказ в статусе "Готовится" (№103).  

**Шаги:**  
1. Открыть раздел "Управление заказами" (или "Заказы").  
2. Нажать фильтр "Ожидают выдачи".  

**ОР:**  
1. В списке отображаются все активные заказы, включая №101, №102, №105 (заказы в работе и ожидающие выдачи).

## Acceptance Criteria 2: Для выбранного заказа предусмотрена кнопка «Отменить заказ».

### Test case 49. Отмена активного заказа
**Тип:** Базовый  
**Предусловие:** Менеджер находится в интерфейсе списка заказов. В списке присутствует заказ №105 со статусом "Ожидает выдачи".  

**Шаги:** 
1. Нажать на строку заказа №105 (или открыть его детальную карточку).  
2. Нажать на кнопку «Отменить заказ».  

**ОР:**  
1. Кнопка «Отменить заказ» отображается.  
2. Заказ отменен

## Acceptance Criteria 3: При нажатии на кнопку система запрашивает подтверждение и предлагает выбрать тип операции: "Освободить стенд", "Вернуть средства".

### Test case 50. Выбор типа операции при отмене заказа
**Тип:** Базовый  
**Предусловие:** Менеджер открыл карточку оплаченного заказа №105 (например, оплата картой).  

**Шаги:**  
1. Нажать кнопку «Отменить заказ».  

**ОР:**  
1. Появляется модальное окно (диалог) с заголовком "Подтверждение отмены".  
2. В диалоге представлены два варианта: «Освободить стенд» и «Вернуть средства»  
3. Кнопки для подтверждения выбора ("Продолжить") и отмены операции.

## Acceptance Criteria 4: Выбрать любой способ возврата средств.

### Test case 51. Возврат средств при отмене заказа
**Тип:** Базовый  
**Предусловие:** В системе есть заказ №106, оплаченный наличными. Менеджер открыл карточку заказа №106 и нажал «Отменить заказ».  

**Шаги:**  
1. В диалоге подтверждения выбрать тип операции «Вернуть средства».  
2. Выбрать "Вернуть наличными".  

**ОР:**  
1. После успешной обработки появляется сообщение "Возврат средств выполнен".  
2. Статус заказа меняется на "Отменён".  
3. Стенд, закреплённый за заказом, освобождается.

## Acceptance Criteria 5: При наличии контактных данных посетителя (телефон) система отправляет уведомление об отмене заказа и, если применимо, возврате средств.

### Test case 52. Отправка SMS-уведомления клиенту об отмене заказа
**Тип:** Базовый  
**Предусловие:** Заказ №108 оплачен, при оформлении указан мобильный телефон клиента (+7 900 123-45-67). Менеджер авторизован и находится в карточке заказа №108.  

**Шаги:**  
1. Нажать кнопку «Отменить заказ».  
2. В диалоге подтверждения выбрать тип операции «Вернуть средства».  
3. Подтвердить отмену.  
4. Дождаться завершения операции возврата средств.  
5. Проверить журнал отправленных уведомлений (или базу данных, лог-файл) на наличие записи об SMS.  

**ОР:**  
1. Заказ успешно отменён, статус изменён на «Отменён», стенд освобождён.  
2. Система отправила SMS на номер +7 900 123-45-67 с текстом, информирующим об отмене заказа и возврате средств (например: «Заказ №108 отменён. Деньги будут возвращены на карту в течение 5–10 рабочих дней»).  
3. В логах системы зафиксировано событие отправки уведомления.  

### Test case 53. Клиент не авторизован в системе
**Тип:** Негативный  
**Предусловие:** Заказ №110 оплачен картой, телефон клиента не указан.  

**Шаги:**  
1. Нажать кнопку «Отменить заказ».  
2. В диалоге подтверждения выбрать тип операции «Вернуть средства».  
3. Подтвердить отмену.  
4. Дождаться завершения операции возврата средств.  
5. Проверить журнал отправленных уведомлений (или базу данных, лог-файл) на наличие записи об SMS.  

**ОР:**  
1. Заказ успешно отменён, статус изменён на «Отменён», стенд освобождён.
2. SMS не отправлен

## Acceptance Criteria 6: Все операции отмены и возврата логируются для последующего аудита.

### Test case 54. Логирование успешной отмены с возвратом на карту
**Тип:** Базовый
**Предусловие:** Менеджер (логин: manager01) выполнил отмену заказа №111 с возвратом средств на карту. Операция прошла успешно.  

**Шаги:**  
1. Перейти в раздел «Журнал аудита» или «Логи операций».  
2. Найти записи, связанные с заказом №111.  
3. Просмотреть детали записи.  

**ОР:**  
1. В логе присутствует запись, содержащая:  
- ID заказа: 111.
- Дата и время операции.
- Тип операции: «Отмена заказа».
- Инициатор: manager01.
- Детали: «Возврат средств на карту, сумма 750.00 руб».
- Результат: «Успешно».

### Test case 55. Логирование отмены с возвратом наличных
**Тип:** Альтернативный  
**Предусловие:** Менеджер (manager02) оформил отмену заказа №112, оплаченного наличными. Операция завершена успешно.  

**Шаги:**
1. Перейти в раздел «Журнал аудита» или «Логи операций».  
2. Найти записи, связанные с заказом №112.  
3. Просмотреть детали записи.  

**ОР:**  
1. В логе присутствует запись, содержащая:  
- ID заказа: 112.  
- Дата и время операции.  
- Тип операции: «Отмена заказа».  
- Инициатор: manager02.  
- Детали: «Возврат наличных средств, сумма 450.00 руб».  
- Результат: «Успешно».  

### Test case 56. Сбой записи в лог (диск полон) – операция отмены не должна выполняться
**Тип:** Негативный  
**Предусловие:** Менеджер (логин: manager04) выполнил отмену заказа №113 с возвратом средств на карту. Операция прошла успешно.  

**Шаги:**  
1. Эмулировать отсутствие свободного места.  
2. Перейти в раздел «Журнал аудита» или «Логи операций».  
3. Найти записи, связанные с заказом №113.  
4. Просмотреть детали записи.  

**ОР:**  
1. Система обнаруживает невозможность записи в лог.  
2. Сообщение: «Невозможно выполнить операцию логирования из-за сбоя системы аудита. Обратитесь к администратору».
