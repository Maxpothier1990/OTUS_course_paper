# User Story 03

| ID | User Story | Приоритет | Версия | Acceptance Criteria |
|----|------------|-----------|--------|---------------------|
| **US03** | Я как **посетитель** хочу **ознакомиться с наполнением корзины**, чтобы **видеть выбранные позиции в своем заказе**. | Высокий | MVP | 1. В корзине отображается список добавленных позиций с фото, названием, количеством, ценой за единицу и общей суммой.<br>2. Итоговая стоимость заказа пересчитывается при добавлении ед. количества товара.<br>3. Итоговая стоимость заказа пересчитывается при удалении ед. количества товара.<br>4. Есть кнопка «Оплатить». |

**Тест-кейсы:**
## Acceptance Criteria 1: В корзине отображается список добавленных позиций с фото, названием, количеством, ценой за единицу и общей суммой.

### Test case 14. Открытие страницы с корзиной с добавленными товарами  
**Тип:** Базовый  
**Предусловие:** Пользователь находится на стартовой странице  

**Шаги:**  
1. Коснуться экрана в любом месте  
2. Выбрать блюдо “Гамбургер”  
3. Нажать на кнопку “Добавить”  
4. Нажать на кнопку “В корзину”   

**OP:**  
1. Открыта страница с корзиной  
2. Добавлен товар “Гамбургер” с фотографией товара, названием, количеством, ценой за единицу  
    
## Acceptance Criteria 2: Итоговая стоимость заказа пересчитывается при добавлении ед. количества товара.

### Test case 15. Пересчет стоимости заказа при добавлении товара
**Тип:** Базовый  
**Предусловие:** Пользователь находится на стартовой странице  

**Шаги:**  
1. Коснуться экрана в любом месте  
2. Выбрать блюдо “Гамбургер”  
3. Нажать на кнопку “Добавить”
4. Нажать на кнопку “В корзину”  
5. В строке с товаром "Гамбургер" нажать на кнопку “Увеличить количество заказываемого блюда на 1 ед.”  

**OP:**  
1. Количество товара “Гамбургер” было X, стало X+1  
2. В поле “сумма к оплате” сумма за товар пересчиталась  
3. В строке товара значение количества обновилось  

### Test case 16. Пересчет стоимости заказа при добавлении этого же товара
**Тип:** Альтернативный  
**Предусловие:** Пользователь находится на стартовой странице  

**Шаги:**  
1. Коснуться экрана в любом месте  
2. Выбрать блюдо “Гамбургер”  
3. Нажать на кнопку “Добавить”  
4. Выбрать этот же товар повторно  
5. Нажать на кнопку “Добавить”  
6. Нажать на кнопку “В корзину”  

**OP:**  
1. Количество товара “Гамбургер” было X, стало X+1  
2. В поле “сумма к оплате” сумма за товар пересчиталась  
3. В строке товара значение количества обновилось  

### Test case 17. Количество товара невозможно увеличить по причине отсутствия единицы товара на складе.
**Тип:** Негативный  
**Предусловие:** Пользователь находится на странице с корзиной. В корзине добавлен товар “Гамбургер”. Количество товара “Гамбургер” = 1 ед  

**Шаги:**  
1. Нажать на кнопку “Увеличить количество заказываемого блюда на 1 ед.”  

**OP:**  
1. Сообщение “Товар отсутствует на складе”

## Acceptance Criteria 3: Итоговая стоимость заказа пересчитывается при удалении ед. количества товара.

### Test case 18. Пересчет стоимости заказа при уменьшении ед. товара
**Тип:** Базовый  
**Предусловие:** Пользователь находится на странице с корзиной. В корзине добавлен товар “Гамбургер”. Количество товара “Гамбургер” = 2 ед  

**Шаги:**  
1. Нажать на кнопку “Уменьшить количество заказываемого блюда на 1 ед”  

**OP:**  
1. Количество товара “Гамбургер” было X, стало X-1  
2. В поле “сумма к оплате” сумма за товар пересчиталась  
3. В строке товара значение количества обновилось  

### Test case 19. Удаление единицы товара в корзине через удаление товара целиком
**Тип:** Альтернативный  
**Предусловие:** Пользователь находится на странице с корзиной. В корзину добавлено 2 единица товара “Гамбургер”. Количество товара “Гамбургер” = 2 ед  

**Шаги:**  
1. Нажать на кнопку “Удалить товар”  

**OP:**  
1. Товар удален из корзины  
2. В поле “сумма к оплате” сумма за товар пересчиталась  
3. В строке товара значение количества обновилось  

### Test case 20. Попытка уменьшить количество товара при отсутствии связи с сервером.
**Тип:** Негативный  
**Предусловие:** Пользователь находится на странице с корзиной. В корзине добавлен товар “Гамбургер”  

**Шаги:**  
1. Эмулировать потерю соединения с интернетом  
2. Нажать на кнопку “Уменьшить количество заказываемого блюда на 1 ед”  

**OP:**  
1. Товар НЕ уменьшен  
2. Появляется сообщение об ошибке: "Не удалось удалить товар. Проверьте подключение к интернету"  
3. Товар "Гамбургер" по-прежнему отображается в корзине

## Acceptance Criteria 4: Если блюдо отсутствует (нет в наличии), то выводится сообщение.
